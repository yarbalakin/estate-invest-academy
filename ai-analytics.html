Новая архитектура
Профили объектов (PropertyCard)
Ключевая идея: для каждого лота с торгов создаём полный профиль из нескольких источников данных. Для каждого аналога -- тоже. Потом сравниваем профиль с профилем по множеству параметров.
Сейчас (простой подход)
  • Берём 50 объявлений по городу и категории
  • Считаем медиану цены за м2
  • ИЖС сравнивается с промкой
  • Центр города сравнивается с окраиной
  • Новостройка сравнивается с развалиной
Новый подход (PropertyCard)
  • Для каждого лота собираем полный профиль из 5+ источников
  • Для каждого аналога -- такой же профиль
  • Сравниваем профиль с профилем: площадь, расстояние, назначение, кадастр
  • Каждый аналог получает оценку похожести (0-100%)
  • Итоговая цена взвешена по похожести

Архитектура
Конвейер обработки (Pipeline)
7 этапов от получения лота до итоговой оценки. Каждый этап обогащает профиль объекта новыми данными.
Этап 1
Сбор лотов
torgi.gov.ru и ЕФРСБ -> парсинг -> базовая карточка с ценой, площадью, адресом и кадастровым номером.
Этап 2
Обогащение карточки лота
Если есть кадастровый номер -- запрос к Росреестру: кадастровая стоимость, категория земли, ВРИ, координаты. Если нет координат -- геокодингГеокодингПреобразование текстового адреса в координаты (широта/долгота). Используем Яндекс.Геокодер или OSM Nominatim (бесплатно). адреса.
Этап 3
Поиск аналогов
На основе обогащённой карточки (не базовой!) ищем аналоги через ads-api.ru с точными фильтрами: категория, ВРИ, тип коммерции, площадь, район, только свежие (90 дней).
Этап 4
Обогащение карточек аналогов
Для каждого аналога: координаты из ads-api, парсинг площади/этажа/типа из параметров. Если есть кадастровый номер в описании -- запрос к Росреестру.
Этап 5
Скоринг похожести
Для каждой пары (лот <-> аналог) считаем оценку похожести по 5 параметрам с весами. Отсекаем аналоги с низким скором.
Этап 6
Взвешенная оценка
Не просто медиана, а взвешенная по похожести: чем больше аналог похож на лот -- тем больше его влияние на итоговую цену. Плюс расчёт confidenceConfidence -- уверенность оценкиПоказывает, насколько можно доверять оценке. Зависит от: количества аналогов с высоким скором, разброса цен между аналогами, полноты данных профиля.Высокий: 8+ аналогов, разброс < 20%
Средний: 4-7 аналогов, разброс 20-40%
Низкий: < 4 аналогов или разброс > 40%.
Этап 7
Результат
Telegram с оценкой -> Google Sheets с полными данными -> SQLite для истории и будущего ML.
Алгоритм
Скоринг похожести
Каждый аналог оценивается по 5 параметрам. Итоговый скор (0-100%) показывает, насколько аналог похож на наш лот. Аналоги с низким скором отбрасываются.
Площадь
25%
Разница площадей в пределах +/-30%. Точное совпадение = 100%, разница > 30% = 0%
Расстояние
25%
Расстояние между лотом и аналогом. < 1 км = 100%, > 10 км = 0%
Назначение
20%
ВРИ совпадает (ИЖС = ИЖС) = 100%. Не совпадает = 30%. Критичный параметр для земли
Кадастровая стоимость
15%
Соотношение кадастровых стоимостей. В пределах +/-50% = пропорционально. Нет данных = нейтральный
Тип объекта
15%
Совпадение типа (квартира = квартира, офис = офис). Совпадает = 100%, нет = 0%

Методология
ФСО-5: Три подхода к оценке
ФСО-5ФСО-5 -- Федеральный стандарт оценки №5Официальный российский стандарт для оценки недвижимости. Обязателен для профессиональных оценщиков. Устанавливает три обязательных подхода к оценке: сравнительный, доходный и затратный. Результаты всех трёх подходов согласовываются для получения итоговой стоимости. -- это государственный стандарт, который определяет, как правильно оценивать недвижимость. Мы используем упрощённую автоматическую версию.
1
Сравнительный
Используем сейчас
  • Находим 5-10 аналогов в радиусе 1-2 км
  • Считаем медиану (или взвешенную) цены за м2
  • Применяем корректировкиКорректировкиПоправки к цене аналога, чтобы учесть различия с нашим объектом. Например: панель вместо кирпича = -7%, 1-й этаж = -5%. Сумма корректировок не должна превышать 15-20%, иначе аналог слишком отличается.Пример: Аналог 4 000 000 руб (кирпич, 5 этаж). Наш объект: панель, 1 этаж.
    Панель: -7%, 1-й этаж: -5% = итого -12%
    4 000 000 x 0.88 = 3 520 000 руб
Самый надёжный для жилья. Работает при наличии аналогов.
2
Доходный
Для коммерции
  • Берём рыночную ставку аренды
  • Вычитаем расходы (налоги, ремонт, простой)
  • Делим на ставку капитализацииСтавка капитализации (Cap Rate)Отношение годового дохода от аренды к стоимости объекта. Показывает доходность инвестиции. Для России: офисы 8-12%, торговля 9-14%, склады 10-15%.Пример: Офис приносит 1.2 млн руб/год аренды. Cap Rate = 10%.
    Стоимость = 1 200 000 / 0.10 = 12 000 000 руб
Лучший для коммерческой недвижимости (офисы, склады).
3
Затратный
Для уникальных объектов
  • Считаем стоимость строительства заново
  • Вычитаем физический износФизический и функциональный износФизический -- разрушение конструкций от времени и эксплуатации. Функциональный -- устаревание планировки, инженерных систем. Например: дому 50 лет, физический износ 40%, функциональный 15%. Итого стоимость = стоимость нового x (1 - 0.40 - 0.15) = 45% от нового. и функциональный
  • Прибавляем стоимость земли
Для объектов без аналогов: заводы, спецздания.
Наша стратегия: начинаем с сравнительного подхода (самый надёжный, самый автоматизируемый). Для коммерции добавляем доходный (аренда). Затратный -- как fallback для уникальных объектов. По ФСО-5 профессиональный оценщик обязан применить все три и согласовать результаты -- мы делаем то же самое, но автоматически.
Технологии
ML-модель для точной оценки
Конечная цель -- обучить модель машинного обученияМашинное обучение (ML)Технология, при которой компьютер учится на данных находить закономерности. Вместо того чтобы программист писал правила "если район X, то цена Y", модель сама находит эти зависимости из тысяч реальных примеров. Чем больше данных -- тем точнее., которая оценивает недвижимость с точностью +/-3-5%.
Сейчас: правила
+/-30-50%
Простая медиана аналогов. Не учитывает десятки факторов. Точность зависит от качества аналогов.
Цель: ML-модель
+/-3-5%
Модель, обученная на тысячах реальных сделок. Учитывает 20+ параметров. Объясняет каждую оценку.
Как это работает
Что такое XGBoostXGBoost -- eXtreme Gradient BoostingАлгоритм машинного обучения, один из самых популярных в мире для табличных данных. Строит сотни "деревьев решений" -- каждое следующее исправляет ошибки предыдущего. Используется в 80% winning-решений на Kaggle (соревнования по ML). Домклик, ЦИАН, Zillow -- все используют похожие алгоритмы.?
  • Алгоритм строит сотни "деревьев решений"
  • Каждое дерево учится на ошибках предыдущего
  • Итог: ансамбль из 500+ деревьев, каждое отвечает за свой аспект оценки
  • Используется Домклик, ЦИАН, Zillow, CoreLogic
SHAPSHAP -- SHapley Additive exPlanationsМетод объяснения предсказаний ML-модели. Показывает вклад каждого фактора в итоговую оценку. Назван в честь нобелевского лауреата Ллойда Шепли. Повышает доверие: вместо "чёрного ящика" пользователь видит прозрачную логику. -- объяснение оценки
  • "Район Мотовилихинский: +420 000 руб"
  • "1-й этаж: -180 000 руб"
  • "Кирпичный дом: +250 000 руб"
  • "Площадь 65 м2: +120 000 руб"
Что нужно для обучения
  • 500-1000 примеров -- минимум для первой модели
  • 2000+ примеров -- для хорошей точности
  • 20+ параметров -- площадь, район, этаж, год, тип, инфраструктура
  • 2-3 месяца сбора -- накапливаем данные в SQLite
Бенчмарки точности

Наша (сейчас, медиана)30-50%
